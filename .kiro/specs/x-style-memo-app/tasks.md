# 実装計画

- [ ] 1. 基盤: データモデル・永続化・共有ユーティリティ
- [x] 1.1 SwiftData データモデル4種と関係・削除規則の定義
  - `Memo` / `Persona` / `Tag` / `ReactionEvent` を `@Model` で定義する
  - 自己参照（親メモ↔返信）と多対多（いいね・タグ）の関係を inverse 明示で張り、`replies` と `ReactionEvent.targetMemo` を cascade、`likedBy`/`tags`/`author`/`replyMemo`/`relatedMemo` を nullify に設定する
  - 画像は外部保存属性、`Tag.name` は一意制約、`createdAt` 不変・`updatedAt` 任意とする
  - 完了状態: 4モデルがコンパイル・永続化でき、親メモ削除で返信が連鎖削除され、いいね・タグ関連は解除されることをサンプルデータで確認できる
  - _Requirements: 7.4, 13.1_
- [ ] 1.2 ModelContainer 構築とアプリエントリポイント
  - アプリ起動時に4モデルを登録した `ModelContainer` をサンドボックス内に構築し、ネットワーク権限・外部同期を一切持たない構成にする
  - 完了状態: アプリが起動し、再起動後もメモが端末内に保持される（オフラインで動作する）
  - _Requirements: 13.1, 13.2, 13.4_
- [ ] 1.3 (P) 共有表示ユーティリティ（相対時刻・アクセントカラー）
  - 日時を「3分前」「昨日」等の相対表現へ整形する純関数と、`accentColor` 文字列を表示色へマッピングするユーティリティを実装する
  - 完了状態: 任意の日時・色名を渡すと期待どおりの表示文字列・色が返る
  - _Requirements: 2.3, 6.3_
  - _Boundary: Shared (RelativeDate, AccentColor)_

- [ ] 2. コア: タグ抽出とメモCRUDサービス
- [ ] 2.1 (P) ハッシュタグ抽出（純関数）
  - 本文から `#` の直後に続く Unicode 文字クラス `\p{L}\p{N}_` の連なりを抽出し、先頭 `#` を除いた重複排除済みのタグ名配列を返す
  - 完了状態: `#メモ` 等の日本語タグ・`#test_1` 等の英数タグ・複数/重複・`#` 単体（無効）の各ケースで期待結果が得られる
  - _Requirements: 4.1, 4.2, 4.6_
  - _Boundary: HashtagExtractor_
- [ ] 2.2 (P) タグの解決・作成（upsert）
  - タグ名配列を受け取り、同名タグが存在すれば再利用、無ければ新規作成して返す
  - 完了状態: 既存タグ名は新規作成されず再利用され、未登録名のみ新規 `Tag` が作られる
  - _Requirements: 4.3, 4.4_
  - _Boundary: TagUpserter_
- [ ] 2.3 (P) Spotlight インデックス管理
  - メモを Core Spotlight の検索可能項目として登録・更新・削除する索引操作を実装する
  - 完了状態: メモ単位で索引の追加・更新・除去が呼び出せ、削除時に対応索引が消える
  - _Requirements: 10.3, 16.5_
  - _Boundary: SpotlightIndexer_
- [ ] 2.4 メモ投稿・編集・削除サービス
  - 投稿: 空白のみの本文を拒否し、本文・作成時刻・親メモ・画像を保存、タグを抽出/upsert して紐づけ、索引へ登録する
  - 編集: 本文・画像を更新し `updatedAt` を設定（`createdAt` 不変）、タグを再抽出して差し替え、索引を再登録、AI 再反応はしない
  - 削除: 通常メモは物理削除し関連を解除、親メモは返信を連鎖削除し、索引を除去する
  - 完了状態: 投稿で新規メモとタグ関連が永続化され、空本文は失敗を返し、編集で `updatedAt` のみ更新され、削除で連鎖・索引整合が保たれる
  - _Requirements: 1.1, 1.4, 1.7, 3.3, 3.4, 4.5, 7.1, 16.2, 16.3, 16.4, 16.5, 16.7_
  - _Depends: 2.1, 2.2, 2.3_

- [ ] 3. コア: 画面（UI）
- [ ] 3.1 コンポーザー（新規・返信・編集）と入力状態

  - 編集モードでは対象メモの本文と画像を初期値として読み込み、返信モードでは親メモIDを保持する
  - 保存はメモサービスへ委譲し、完了時に直前の画面へ自動的に戻る
  - 完了状態: 新規・返信・編集の各モードで入力・画像添付・保存・自動復帰が動作し、空本文では投稿操作が無効化される

- [ ] 3.2 (P) タイムライン画面とメモ行表示

  - FAB から新規コンポーザー、行タップでスレッド詳細へ遷移、行スワイプで確認ダイアログ経由の削除を行う
  - 完了状態: 数千件規模でも降順一覧が滑らかにスクロールし、各情報が表示され、FAB/タップ/スワイプ削除が機能する
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 2.5, 2.6, 2.7, 2.8, 3.1, 3.2, 14.1, 14.2, 16.6_

- [ ] 3.3 (P) スレッド詳細画面

  - ペルソナの返信は名前とアクセントカラーで自分の返信と区別し、AI による反応であることを明示する
  - 「返信」導線から親メモIDを渡した返信モードのコンポーザーを開き、親メモのスワイプ削除は確認ダイアログを挟む
  - 完了状態: 親＋返信が正しい順で表示され、ペルソナ返信が識別表示され、返信導線と削除確認が機能する

  - 完了状態: 本文検索・タグ絞り込みのいずれでも降順の結果が得られる

- [ ] 5. コア: AI ペルソナ・リアクション・関連メモ（RAG）
- [ ] 5.1 (P) プリセットペルソナの初回シード

- [ ] 5.2 (P) 構造化出力スキーマの定義

  - 完了状態: 2種の生成スキーマが定義され、後続のエンジンから参照できる
  - _Requirements: 9.1, 10.2_
  - _Boundary: PersonaReaction (PersonaReaction, PersonaInsight)_
- [ ] 5.3 (P) 関連メモ検索（経路切替）
  - 経路非依存の関連メモ検索インターフェースと、Spotlight 経路（対応端末）・タグ重複スコア・本文埋め込み類似度のフォールバック実装を用意し、起動時に適切な実装を注入する
  - 返却メモは実在し（grounded）対象自身を含まないことを保証する
  - 完了状態: いずれの経路でも対象メモに関連する過去メモが関連度順で返り、対象自身が除外される

  - _Boundary: RelatedMemoFinder, SpotlightIndexer_
  - _Depends: 2.3_
- [ ] 5.4 ペルソナリアクションエンジン
  - 背景アクターの専用コンテキストで、(1) 関連メモをメモ単位に1回だけ算出し全ペルソナで共有 →(2) 各ペルソナを遅延を挟み逐次/低並行で生成・適用 →(3) 関連メモがある場合のみ代表1体が気づきを1回生成、の順で処理する
  - 各ペルソナの性格指示文で独立セッションを張り、関連メモを文脈に与えて grounded 化し、いいねは `likedBy` 追加＋イベント記録、返信は返信メモ作成＋イベント記録、気づきはイベント記録する
  - 個別の生成失敗はスキップし、推論は端末内で完結させ UI をブロックしない
  - 完了状態: 投稿IDを渡すと背景でいいね/返信/気づきが適用され、対応するリアクションイベントが記録され、UI が固まらない
  - _Requirements: 8.3, 8.4, 9.1, 9.2, 9.3, 9.4, 9.5, 9.6, 10.2, 10.5, 11.2, 13.3, 15.4, 15.5_
